How to Set Up Jellyfin with NVIDIA GPU Transcoding (GTX 1070 — Fix AV1 Errors)

📅 June 2026 ⏱ ~10 min read OS: Windows 11 Jellyfin NVENC GTX 1070 4K UHD

If you've set up Jellyfin with an NVIDIA GTX 1070 (or similar older GPU) and your 4K streams are stuttering or failing, there's a good chance Jellyfin is trying to transcode to AV1 — which the GTX 1070 can't do. This guide walks through the full setup: ripping 4K Blu-rays with MakeMKV, flashing your ASUS drive for UHD support, and configuring Jellyfin's NVENC settings so it actually uses the GPU.

🚫
The problem: Jellyfin defaults to AV1 encoding. AV1 NVENC requires an RTX 4000 series GPU or newer. The GTX 1070 only supports H264 and HEVC via NVENC — if you don't disable AV1, transcoding will fail or fall back to slow software encoding.

Ripping 4K Blu-rays with MakeMKV

Before you can stream anything, you need to rip your discs to MKV files. The ASUS BW-16D1X-U can rip standard Blu-rays out of the box, but 4K UHD requires a firmware flash first.

Flashing the ASUS drive for 4K UHD

1

Download the tools

From the MakeMKV forum, download MakeMKV, the SFD Tool, and the "All You Need" firmware pack. All three are needed.

2

Flash the firmware (Windows only)

Use SFD Tool to flash ASUS-BW-16D1HT-3.10-WM01601.bin onto the drive. The SFD Tool is a .exe — this step is confirmed on Windows. Mac and Linux may work but are untested.

3

Rip with MakeMKV

After flashing, insert a 4K UHD disc and open MakeMKV. It should detect the disc and let you rip it to MKV. Without the flash, only standard 1080p Blu-rays will work.

⚠️
If you don't need 4K UHD, skip the firmware flash entirely. The drive works perfectly for 1080p Blu-ray ripping with no modifications.

Configuring Jellyfin NVENC transcoding

This is where most people get stuck. Jellyfin's default settings will try AV1 encoding, which doesn't work on the GTX 1070. Here's exactly what to set.

Go to Jellyfin Dashboard → Admin → Playback → Transcoding and apply the following:

Hardware acceleration

Set Hardware Acceleration to Nvidia NVENC. You'll see a note that additional configuration is required — that's fine, we're handling it below.

Hardware decoding — enable everything except AV1

Save the settings. If everything's correct, when you play a 4K file that needs transcoding, Task Manager will show the GTX 1070 with active Video Encode and Video Decode usage. The Jellyfin dashboard will show something like: 135fps · 39.9 Mbps · MP4 HEVC AAC.

Client compatibility

Not all Jellyfin clients behave the same way when it comes to transcoding.

Sony TV (Android TV) — Direct Play

Install the Jellyfin app from the Google Play Store on your Sony TV. Enter your server's IP address, log in, and your library appears. If the TV and server are on the same network or VLAN, the TV may auto-detect the server. When the TV is connected via ethernet, 4K UHD files play as Direct Play — the raw file streams directly with zero transcoding, which is the ideal scenario.

Windows browser or app — NVENC transcoding

Windows clients hit the server and trigger GPU transcoding when the video format or bitrate exceeds what the client can handle directly. With NVENC configured correctly, the GTX 1070 handles it cleanly.

macOS — no NVIDIA support

⚠️
NVIDIA NVENC transcoding does not work on macOS. This is an OS-level limitation — Apple dropped NVIDIA support years ago. If you're using a web browser or the Jellyfin Mac app on a Mac, it will not use your server's GPU. It falls back to local hardware decode using Apple's own video engine instead. The transcoding still happens, it just doesn't use your GTX 1070.

Troubleshooting

GPU isn't being used — still on CPU

Check that AV1 encoding is disabled (both in hardware decoding list and encoding format options). Also confirm the NVIDIA drivers are up to date. Open Task Manager → Performance → GPU 1 and watch the Video Encode/Decode graphs while playing something that requires transcoding.

Getting decoding errors

If you see errors in the Jellyfin log related to decoding, try disabling Enhanced NVDEC decoder in the transcoding settings. This switches to CUVID instead, which is more compatible on some driver versions.

Transcoding reason: "video's bitrate exceeds the limit"

This is normal — it means the file's native bitrate is higher than the client's playback limit, so Jellyfin is re-encoding it at a lower bitrate. As long as the GPU is doing the work, this is fine.

ℹ️
If you want AV1 support in the future, you'll need an RTX 4000 series card or newer. The GTX 1070 tops out at HEVC (H.265) encoding via NVENC.
